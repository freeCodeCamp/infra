## What this delivers

Closes the cassiopeia operator-env half that was assumed pre-existing
but never actually wired. After D41/D42 collapsed the per-cluster R2
ops cred, this dispatch shrinks to two mechanical actions:

1. Export `R2_BUCKET=universe-static-apps-01` from
   `k3s/gxy-cassiopeia/.envrc` so the smoke recipe + future tooling
   resolve the bucket name without inline literals.
2. Pull cassiopeia kubeconfig into operator's `~/.kube/config` (for
   failure diagnosis on smoke red — `kubectl -n caddy logs ...`).

**Single-bucket invariant** reinforced: the `R2_BUCKET` export carries
the only bucket name. No per-site / per-cluster buckets.

## Operator steps

### 1. Patch `k3s/gxy-cassiopeia/.envrc`

Current content:

```bash
source_env ../../.envrc
if [ -d "$SECRETS_DIR" ]; then
  use_sops "$SECRETS_DIR/do-universe/.env.enc"
fi
export KUBECONFIG="$(expand_path .kubeconfig.yaml)"
dotenv_if_exists .env
```

Add the `R2_BUCKET` export. Final content:

```bash
source_env ../../.envrc
if [ -d "$SECRETS_DIR" ]; then
  use_sops "$SECRETS_DIR/do-universe/.env.enc"
fi
export KUBECONFIG="$(expand_path .kubeconfig.yaml)"
export R2_BUCKET=universe-static-apps-01
dotenv_if_exists .env
```

Note: NO `use_sops` for any cassiopeia-scoped `.env.enc` — admin S3
ops keys live in `windmill/.env.enc` and are sops-decrypted on demand
by the smoke script (D41). `caddy.values.yaml.enc` stays where it is
(consumed by Helm at deploy time, not by direnv shell).

After edit:

```bash
cd ~/DEV/fCC/infra/k3s/gxy-cassiopeia
direnv allow
echo "R2_BUCKET=${R2_BUCKET}"   # → universe-static-apps-01
```

### 2. Pull cassiopeia kubeconfig

```bash
doctl kubernetes cluster kubeconfig save gxy-cassiopeia-k3s \
  --merge --set-current-context=false

# Or, if k3s is hand-rolled and not DOKS:
# scp <node>:/etc/rancher/k3s/k3s.yaml ~/.kube/cassiopeia.yaml
# (then merge / select via KUBECONFIG)

# Verify access:
kubectl --context gxy-cassiopeia get nodes
```

If `--set-current-context=false` doesn't preserve cassiopeia naming,
operator picks the appropriate flag for their local kubeconfig hygiene.

### 3. Stage `.envrc` change

```bash
cd ~/DEV/fCC/infra
git add k3s/gxy-cassiopeia/.envrc
# defer commit until G1.1.smoke closes (single closure commit covers both)
```

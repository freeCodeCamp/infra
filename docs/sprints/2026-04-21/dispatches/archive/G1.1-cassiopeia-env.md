## What this delivers

Closes the cassiopeia operator-env half that was assumed pre-existing
but never actually wired. After D41/D42 collapsed the per-cluster R2
ops cred, this dispatch shrinks to two mechanical actions:

1. Export `R2_BUCKET=universe-static-apps-01` from
   `k3s/gxy-cassiopeia/.envrc` so the smoke recipe + future tooling
   resolve the bucket name without inline literals.
2. Sanity-check that the existing per-cluster kubeconfig at
   `k3s/gxy-cassiopeia/.kubeconfig.yaml` still works against the
   cluster (Tailscale-routed `100.64.x.x:6443`) — used for failure
   diagnosis on smoke red (`kubectl -n caddy logs ...`).

**Single-bucket invariant** reinforced: the `R2_BUCKET` export carries
the only bucket name. No per-site / per-cluster buckets.

## Tooling convention (read before steps)

This repo's k3s clusters are **hand-rolled** (cloud-init bootstrap on DO
droplets joined into a Tailscale tailnet) — **NOT** DigitalOcean
Kubernetes (DOKS). Therefore:

- `doctl kubernetes cluster kubeconfig save ...` is the **wrong tool**
  (DOKS-only) — use the patterns below.
- Per-cluster kubeconfig lives at `k3s/<cluster>/.kubeconfig.yaml`,
  loaded by direnv via `export KUBECONFIG="$(expand_path .kubeconfig.yaml)"`.
- Operator kubectl invocation pattern: `cd k3s/<cluster>` (direnv
  loads KUBECONFIG) **or** `direnv exec k3s/<cluster> kubectl ...`
  from outside. **No `kubectl --context <cluster>`** — there is no
  named context, the kubeconfig has a single default cluster.
- Restoration recipe: `just kubeconfig-sync <cluster>` decrypts
  `${SECRETS_DIR}/k3s/<cluster>/kubeconfig.yaml.enc` to local. NOTE:
  `infra-secrets/k3s/gxy-cassiopeia/kubeconfig.yaml.enc` does not yet
  exist (file once-only seeded the local `.kubeconfig.yaml` from a node
  via `scp /etc/rancher/k3s/k3s.yaml`); seeding it is a TODO-park
  candidate, not in scope here.
- Cluster reach requires Tailscale up — server URL is
  `https://100.64.205.100:6443` (a `tailnet` IP).

fi

Add the `R2_BUCKET` export. Final content:

```bash
source_env ../../.envrc
if [ -d "$SECRETS_DIR" ]; then
  use_sops "$SECRETS_DIR/do-universe/.env.enc"
fi
export KUBECONFIG="$(expand_path .kubeconfig.yaml)"
export R2_BUCKET=universe-static-apps-01
dotenv_if_exists .env
```

Note: NO `use_sops` for any cassiopeia-scoped `.env.enc` — admin S3
ops keys live in `windmill/.env.enc` and are sops-decrypted on demand
by the smoke script (D41). `caddy.values.yaml.enc` stays where it is
(consumed by Helm at deploy time, not by direnv shell).

After edit:

```bash
cd ~/DEV/fCC/infra/k3s/gxy-cassiopeia
direnv allow
echo "R2_BUCKET=${R2_BUCKET}"   # → universe-static-apps-01
```

### 2. Sanity-check cassiopeia kubeconfig (no pull needed)

The per-cluster kubeconfig already exists at
`k3s/gxy-cassiopeia/.kubeconfig.yaml` (seeded once from a node via
`scp /etc/rancher/k3s/k3s.yaml`; server URL = Tailscale-routed
`https://100.64.205.100:6443`). Verify reachability:

```bash
# From outside the cluster dir:
direnv exec ~/DEV/fCC/infra/k3s/gxy-cassiopeia kubectl get nodes

# Or after `cd k3s/gxy-cassiopeia` (direnv loads KUBECONFIG):
kubectl get nodes
```

Expected: 3 nodes Ready (`gxy-vm-cassiopeia-k3s-{1,2,3}`).

If kubectl fails:

- **`Unable to connect to the server: dial ...`** → Tailscale down on
  operator machine. Run `tailscale up`, retry.
- **`x509: certificate ... not yet valid` / cert errors** → kubeconfig
  stale (cert rotated). Reseed from a node:
  `scp <node>:/etc/rancher/k3s/k3s.yaml k3s/gxy-cassiopeia/.kubeconfig.yaml`
  then `chmod 600` and rewrite the `server:` line to the Tailscale IP.
- **File missing entirely** → currently no `kubeconfig.yaml.enc` in
  `infra-secrets/k3s/gxy-cassiopeia/`, so `just kubeconfig-sync` won't
  help. Reseed via `scp` as above and consider filing a TODO-park to
  encrypt + commit a canonical copy under infra-secrets.

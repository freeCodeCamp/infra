### 4. Push Windmill Resource `u/admin/cf_r2_provisioner`

T11 dispatch ¬ß6 specifies shape `{cfApiToken, cfAccountId}` (Bearer +
Account ID only ‚Äî no S3 keys in this Resource; S3 keys live separately
in `windmill/.env.enc` for direct script consumption).

```bash
cd ~/DEV/fCC-U/windmill

# Decrypt admin file to a process-local var (no shell persist):
ADMIN_ENV="$(sops -d --input-type dotenv --output-type dotenv ~/DEV/fCC/infra-secrets/windmill/.env.enc)"
TOKEN="$(printf '%s\n' "$ADMIN_ENV" | awk -F= '$1=="CF_R2_ADMIN_API_TOKEN"{print substr($0, index($0,"=")+1); exit}')"
ACCT="$(printf '%s\n' "$ADMIN_ENV" | awk -F= '$1=="CF_ACCOUNT_ID"{print substr($0, index($0,"=")+1); exit}')"

# Verify both vars are non-empty before push:
[[ -n "$TOKEN" && -n "$ACCT" ]] || { echo "FAIL: admin vars empty"; return 1; }

# Resource type may need to exist first. Check:
bunx wmill resource list-types --workspace platform | grep -E '^c_cf_r2_provisioner' \
  || echo "NOTE: resource-type c_cf_r2_provisioner not yet defined ‚Äî push may auto-create or require type definition first"

# Push:
bunx wmill resource push u/admin/cf_r2_provisioner \
  --workspace platform \
  --resource-type c_cf_r2_provisioner \
  --value "$(jq -nc --arg t "$TOKEN" --arg a "$ACCT" '{cfApiToken:$t, cfAccountId:$a}')"
```

If `c_cf_r2_provisioner` resource type doesn't exist, create it via
Windmill UI first (Resources ‚Üí Resource Types ‚Üí Add custom type, two
fields: `cfApiToken: string`, `cfAccountId: string`) ‚Äî then re-run
push.

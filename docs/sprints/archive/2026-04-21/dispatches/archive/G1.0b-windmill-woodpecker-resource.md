## What this delivers

T11 windmill flow's second admin-side dependency: the Woodpecker admin
Resource that lets the flow register repo-scoped per-site secrets via
the Woodpecker REST API.

Audit found this work was never started. T11 cannot register
per-site Woodpecker secrets without it.

Live-verified facts (audit 2026-04-25):

- Woodpecker host: `https://woodpecker.freecodecamp.net` (200, version 3.13.0)
- **API base: `/api`** (NOT `/api/v1`). `/api/user` returns 401
  (real auth surface). `/api/v1/...` returns 200 HTML (SPA fallback).

## Operator steps

### 1. Mint Woodpecker personal access token

- Browser → `https://woodpecker.freecodecamp.net`
- Sign in (GitHub OAuth via `freeCodeCamp-Universe` org)
- User menu → User settings → Personal access tokens → New token
- Name: `windmill-platform-admin`
- Scope: admin (org-scope sufficient — repo-scope token cannot
  enumerate `freeCodeCamp-Universe/*` repos for secret registration)
- Expiry: 1 year (or "no expiry" if UI permits — rotation runbook TBD)
- Copy token immediately (cannot view again)

### 2. Confirm API base + smoke probe (read-only)

```bash
# Should return 401 — confirms `/api` is the real REST surface
curl -sS -o /dev/null -w "/api/user no-auth -> %{http_code}\n" \
  https://woodpecker.freecodecamp.net/api/user

# With token, should return 200 + JSON user object
curl -sS -H "Authorization: Bearer <paste-token-from-step-1>" \
  https://woodpecker.freecodecamp.net/api/user | jq '.login // .id'

# Should list freeCodeCamp-Universe repos:
curl -sS -H "Authorization: Bearer <token>" \
  https://woodpecker.freecodecamp.net/api/repos | jq '.[].full_name' | grep freeCodeCamp-Universe | head -3
```

If `/api/user` with token doesn't return JSON — STOP, escalate.

### 3. Create resource type `c_woodpecker_admin` (if not present)

Windmill UI → Workspace `platform` → Resources → Resource Types → Add:

- Name: `c_woodpecker_admin`
- Schema:
  - `baseUrl: string` (e.g. `https://woodpecker.freecodecamp.net/api`)
  - `token: string` (the PAT minted in step 1)

Or via wmill (if the type isn't already defined):

```bash
cd ~/DEV/fCC-U/windmill
bunx wmill resource list-types --workspace platform | grep '^c_woodpecker_admin' \
  || echo "type not defined — create via UI or via resource-type push"
```

### 4. Push Resource `u/admin/woodpecker_admin`

```bash
cd ~/DEV/fCC-U/windmill

# Confirm token + base captured into temporary vars (do not echo):
read -r -s -p "Paste Woodpecker token: " WP_TOKEN; echo
WP_BASE="https://woodpecker.freecodecamp.net/api"   # NO trailing slash

bunx wmill resource push u/admin/woodpecker_admin \
  --workspace platform \
  --resource-type c_woodpecker_admin \
  --value "$(jq -nc --arg b "$WP_BASE" --arg t "$WP_TOKEN" '{baseUrl:$b, token:$t}')"

unset WP_TOKEN
```

## Acceptance criteria

- Woodpecker PAT exists with admin scope sufficient to enumerate
  `freeCodeCamp-Universe/*` repos and POST per-repo secrets
- Resource type `c_woodpecker_admin` exists in platform workspace
- Windmill Resource `u/admin/woodpecker_admin` exists with shape
  `{baseUrl: "https://woodpecker.freecodecamp.net/api", token: <non-empty>}`
- `baseUrl` ends with `/api`, NO trailing slash, NO `/v1` suffix

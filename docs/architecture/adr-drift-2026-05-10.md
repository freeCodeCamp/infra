### ADR-002 — IaC Tooling (drift, S3)

**Claim:** OpenTofu + Ansible + Atlantis. OpenTofu manages galaxy VMs, DNS, firewalls, R2 state.

**Reality:**

- Ansible **live** — `ansible/playbooks/play-k3s--*.yml` bootstrap k3s on every galaxy. ✓
- OpenTofu **not adopted** — no `terraform/` workspace exists for any of the three live galaxies. cassiopeia flight-manual line 33 (current) explicitly: _"No OpenTofu workspace exists for gxy-cassiopeia yet — provisioning is DO dashboard today, codify as a follow-up when the layout stabilises post-cutover."_
- Atlantis **parked** per spike-plan §"What NEVER moves" + TODO-park §"Atlantis".
- Cloudflare R2 backend for Tofu state — moot until Tofu workspace exists.

**Remediation:**

- Amend ADR-002 Status: `Accepted (Ansible Phase-1 live; OpenTofu+Atlantis adoption deferred — parked behind first multi-galaxy provisioning need)`.
- Move "What OpenTofu manages" table behind a "Future state" header.
- No code change required.

### ADR-016 — Deploy Proxy Plane (drift, S1)

The cassiopeia-anchoring ADR. Drift surfaces below are S1 because they affect the operator runbook the user is going to rehearse.

- ADR Q15 says: _"New repo `freeCodeCamp/uploads` at `~/DEV/fCC-U/uploads/`. Go module `github.com/freeCodeCamp/uploads`."_
- Reality: repo is `freeCodeCamp/artemis` at `~/DEV/fCC/artemis/`. Service was renamed `uploads → artemis` post-decision. Existing flight-manual, chart, and field-notes refer to "artemis"; only ADR-016 still says "uploads".

**Remediation:**

- Amend ADR-016 Q15 to record the rename and final canonical repo path.
- Update §Architecture diagram label `uploads.freecode.camp` → no change (the public hostname **is** `uploads.freecode.camp`; the **service name** is `artemis`). Add an explicit note distinguishing the two.

**Drift 2 — sites.yaml lifecycle assumes ArgoCD sync:**

- ADR §"sites.yaml lifecycle" line 181: _"ArgoCD syncs ConfigMap; pod fsnotify-reloads without restart."_
- Reality: ArgoCD parked. Today's lifecycle: PR merge to `freeCodeCamp/artemis` → operator pulls latest → re-runs `just release gxy-management artemis` (which embeds sites.yaml via `--set-file`) → ConfigMap mutates → fsnotify reloads in-pod.

**Remediation:**

- Amend §"sites.yaml lifecycle" to record the **current** operator-driven flow as authoritative; note the ArgoCD-driven flow as the post-D005 reactivation target.
- Note that this drift is the **driver** for the C3 finding (operator/staff workflow coupling). T3 RFC redesigns the registry to remove the coupling regardless of when ArgoCD reactivates.

- Per ADR README "Broken-ref note (2026-05-04)": ADR-016:236-237 references the archived RFC `rfc-gxy-cassiopeia.md`.
